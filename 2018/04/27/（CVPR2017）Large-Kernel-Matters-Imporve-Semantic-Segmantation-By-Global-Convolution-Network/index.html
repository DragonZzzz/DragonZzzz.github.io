（CVPR2017）Large Kernel Matters-Imporve Semantic Segmantation By Global Convolution Network

（CVPR2017）Large Kernel Matters-Imporve Semantic Segmantation By Global Convolution Network

Face++在CVPR2017年的工作，在PASCAL VOC 2012上有82.2%的效果。

通过最近的很多Semantic Segmentation的模型我们可以发现，小的卷积核广泛地用于这些模型中，因为大量小的卷积核能够拟合大卷积核的效果，但是在计算上的开销要小的多。 但是本文中，作者从localization和classification两个方面阐述，大的卷积核可以提供更好的效果。不同于直接使用大的卷积核，作者提出了Global Convolution Network，保证了计算上的开销相对于直接使用大卷积核足够小。 同时，一个简单的residual-based boundaries refinement 用于优化分割的边缘。

Global Convolution Network

还是讨论语义分割中的两个主要问题：

  • localization : fully convolutional and no fully-connected and pooling， transformation-sensitive
  • classification : 需要网络中存在densely connections用于获取global information，从而实现transformation-invariant
    可以说，这两个问题本身就是矛盾的，Global Convolution Network就是想同时处理这两种问题。

Vgg等结构最后使用fc或者global pooling使得features对于干扰很鲁棒，并且对于变换不敏感。
最近的语义分割结构考虑到spatial resolution的问题，使用的Deconv，dilated convolution等方法。

现在的语义分割方法只考虑了localization问题而忽略了classification问题，使得网络对于input不鲁棒，并且网络的Effective Receptive Field并不能像理论的一样覆盖整幅图像。
如下图所示，对于A图片，网络的RF能够很好的感知整个鸟，而将input做一个变换为图B之后，网络的RF则不能很好的感知整个鸟，影响了网络的识别效果。

从localization的角度来看，网络中不能含有fc层和glocal pooling层，而从classification的角度看，网路走红的卷积层的kernel需要足够的大用于提供巨大的RF，这就时网络设计的两个原则，基于这两个原则，提出了GCN。

利用这样一个结构拟合大的卷积核，并且能够明显的减小计算上的开销。相比于k*k的卷积核，GCN只有$O(2/k)$的开销和参数数量。
